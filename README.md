# -Python-ML-
📌 项目概述
本项目旨在通过机器学习算法识别电子支付中的非法转账行为。基于 PaySim 模拟数据集（630万条交易记录），构建了一套整合了监督学习（随机森林、决策树）与无监督学习（孤立森林）的混合风控系统。项目重点解决了金融数据中极度不平衡的样本分布问题，并设计了可落地的阶梯式验证风控策略。

🛠️ 技术栈
编程语言: Python (Pandas, NumPy, Scikit-learn)

核心算法: 随机森林 (Random Forest), 决策树 (Decision Tree), 孤立森林 (Isolation Forest), Apriori 关联规则

数据工程: SMOTE (过采样技术), 特征工程, 降维分析

🔍 核心技术亮点
不平衡数据处理 (Imbalanced Data): 针对欺诈样本仅占 0.1% 的极端情况，采用 SMOTE 技术生成合成样本，显著提升了模型对少数类（欺诈行为）的捕捉能力。

混合建模架构:

监督学习: 利用随机森林实现对已知欺诈模式的精准识别（F1-Score 达到 1.0）。

无监督学习: 引入孤立森林用于识别未知的异常交易模式，增强系统对新型欺诈的泛化能力。

特征重要性挖掘: 识别出 amount (交易金额) 和 oldbalanceOrg (原始余额) 是预警欺诈的关键指标。

📈 业务影响与决策支持 (Decision Analytics)
财务减损: 经模型验证测算，该系统预计可为机构避免约 14.8 万美元 的潜在欺诈损失。

零误报策略: 通过设计阶梯式身份验证（Step-up Authentication），系统仅对高风险交易触发二次验证，确保了 99.9% 正常用户的支付体验。
